# Google Sheets 공용 저장 설정

> 이전 Google Sheets + Apps Script 방식 안내입니다. 현재 공용 저장은 `FIREBASE_SETUP.md`의 Firebase Firestore 방식이 기준입니다.

이 앱은 기본적으로 기존 `localStorage` 데이터를 삭제하지 않습니다. Google Sheets 공용 저장을 사용하려면 아래 순서대로 설정하세요.

## 1. Google Sheet 만들기

1. 새 Google 스프레드시트를 만듭니다.
2. 메뉴에서 `확장 프로그램 > Apps Script`를 엽니다.
3. 이 저장소의 `google-apps-script/Code.gs` 내용을 Apps Script 편집기에 붙여넣고 저장합니다.
4. 외부 접근을 제한하려면 `Code.gs` 상단의 `API_TOKEN`에 임의의 긴 문자열을 입력합니다.

## 2. 웹앱 배포

1. Apps Script 상단의 `배포 > 새 배포`를 클릭합니다.
2. 유형은 `웹 앱`을 선택합니다.
3. 실행 권한은 본인 계정으로 설정합니다.
4. 액세스 권한은 운영 방식에 맞춰 설정합니다.
   - 회사 내부 Google 계정만 쓸 경우: 조직 사용자
   - GitHub Pages에서 모든 직원이 접근해야 할 경우: 링크가 있는 사용자 또는 모든 사용자
5. 배포 후 발급되는 `/exec` URL을 복사합니다.

## 3. 앱에서 연결

1. HNMT COIL 웹앱에 로그인합니다.
2. 상단 `공용저장` 버튼을 누릅니다.
3. Apps Script 웹앱 URL을 입력하고 `URL 저장`을 누릅니다.
4. `Code.gs`에 `API_TOKEN`을 설정했다면 같은 값을 `공용 저장 API 토큰`에 입력합니다.
5. 먼저 `기존 데이터 JSON 백업`을 눌러 백업 파일을 내려받습니다.
6. `로컬 데이터를 Sheets로 이전`을 누르면 현재 브라우저 데이터를 Google Sheets로 저장합니다.

## 데이터 안전 규칙

- 기존 localStorage 데이터는 자동 삭제하지 않습니다.
- 시트의 1행은 컬럼명으로 사용합니다.
- 요청된 6개 탭(`색상마스터`, `기초재고`, `출고내역`, `예약내역`, `변경타임라인`, `설정`)만 생성합니다.
- 기존 앱 호환에 필요한 상세 스냅샷은 `설정` 탭에 JSON 조각으로 보존합니다.
- 컬럼 순서가 바뀌어도 컬럼명 기준으로 읽고 저장합니다.
- 필요한 컬럼이 없으면 자동으로 추가합니다.
- 앱이 모르는 추가 컬럼은 삭제하지 않고 보존합니다.
- 초기화는 운영 데이터만 대상으로 하며 색상마스터 기본 목록은 유지합니다.
- 저장 전 최신 Sheets 데이터를 다시 읽고 병합한 뒤 저장합니다.
- Apps Script는 `LockService`와 `updatedAt` 충돌 감지로 동시 저장 충돌을 재시도하게 합니다.
- 초기화 실행 전 현재 브라우저 데이터 JSON 백업을 자동으로 내려받습니다.
